#### 缓存策略

| 策略 | 说明 | 应用场景 |
|------|------|----------|
| Cache-Aside | 先查缓存，未命中再查数据库 | 帖子详情、用户统计 |
| Write-Through | 写入时同时更新缓存 | 关注/取消关注 |
| TTL过期 | 设置过期时间自动失效 | 热门声音、帖子列表 |
| 主动清除 | 数据变更时清除相关缓存 | 用户信息更新 |

#### 缓存键设计

```
# 用户相关
user:follows:{userId}           # 关注列表 (TTL: 5min)
user:followers:{userId}         # 粉丝列表 (TTL: 5min)
user:stats:{userId}             # 统计数据 (TTL: 1min)

# 内容相关
post:detail:{postId}            # 帖子详情 (TTL: 5min)
posts:list:{page}:{limit}       # 帖子列表 (TTL: 1min)
sounds:popular                  # 热门声音 (TTL: 10min)

# 消息相关
messages:unread:{userId}        # 未读数 (TTL: 30sec)
follow:stats:{userId}           # 关注统计 (TTL: 1min)
```

#### 消息类型

| 类型 | 方向 | 说明 |
|------|------|------|
| `ping/pong` | C↔S | 心跳检测 |
| `chat_message` | C→S | 发送消息 |
| `new_message` | S→C | 收到新消息 |
| `typing` | C↔S | 打字状态 |
| `mark_read` | C→S | 标记已读 |
| `message_read` | S→C | 消息已读通知 |
| `user_online` | S→C | 用户上线通知 |
| `user_offline` | S→C | 用户下线通知 |
| `unread_count` | S→C | 未读数更新 |

#### 连接管理

```javascript
// 连接存储
const connectedUsers = new Map();
// Key: userId (String)
// Value: WebSocket Socket对象

// 功能
- 用户上线: connectedUsers.set(userId, socket)
- 用户下线: connectedUsers.delete(userId)
- 消息推送: connectedUsers.get(receiverId).write(frame)
- 广播通知: connectedUsers.forEach((socket, id) => ...)
```

### 6.2 验证码机制

#### 验证码流程

```
┌─────────────────────────────────────────────────────────────┐
│                    验证码安全机制                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 生成验证码                                               │
│     ┌─────────────┐    ┌─────────────┐    ┌─────────────┐  │
│     │ 随机生成4位  │───▶│ 生成SVG图片  │───▶│ 存储到内存  │  │
│     │ 字母数字    │    │ Base64编码  │    │ Map<token>  │  │
│     └─────────────┘    └─────────────┘    └─────────────┘  │
│                                                             │
│  2. 验证流程                                                 │
│     ┌─────────────┐    ┌─────────────┐    ┌─────────────┐  │
│     │ 用户输入    │───▶│ 检查token   │───▶│ 比对验证码  │  │
│     │ 验证码     │    │ 是否存在    │    │ 是否正确    │  │
│     └─────────────┘    └─────────────┘    └─────────────┘  │
│                              │                   │          │
│                              ▼                   ▼          │
│                        ┌─────────────┐    ┌─────────────┐  │
│                        │ 不存在/过期  │    │ 验证成功    │  │
│                        │ 返回错误    │    │ 立即删除    │  │
│                        └─────────────┘    └─────────────┘  │
│                                                             │
│  安全特性:                                                   │
│  • 5分钟有效期                                              │
│  • 一次性使用（验证后立即删除）                              │
│  • 30秒定时清理过期验证码                                    │
│  • 16字节随机token                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 应用场景

- **管理员登录**: 必须携带验证码才能登录
- **防止暴力破解**: 增加自动化攻击成本
- **人机验证**: 区分真实用户和机器人

### 6.3 安全措施

| 措施 | 说明 |
|------|------|
| 密码加密 | 使用 bcrypt 进行单向哈希加密 |
| JWT认证 | Token有效期7天，包含用户ID和权限信息 |
| 管理员验证 | 单独的管理员登录接口，验证 is_admin 字段 |
| 验证码机制 | 管理员登录需要图形验证码验证 |
| SQL注入防护 | 使用参数化查询 |
| 文件上传限制 | 限制文件类型和大小 |
| CORS配置 | 限制跨域访问来源 |

## 十、总结

本项目采用前后端分离架构，前端使用 UniApp 跨端框架，后端使用 Node.js + Express + MySQL 技术栈。系统主要功能包括声音识别分类、社区互动、私信聊天等模块。

### 技术亮点

1. **跨端开发**: UniApp 一套代码支持微信小程序、H5、iOS、Android 四端
2. **双入口架构**: 独立页面 + 整合入口，兼顾直接访问和统一体验
3. **Redis 缓存**: 多级 TTL 缓存策略，显著提升查询性能
4. **WebSocket 实时通信**: 自定义实现，支持实时聊天、在线状态、消息推送
5. **验证码机制**: 管理员登录安全验证，防止暴力破解
6. **软删除机制**: 聊天记录删除不影响对方
7. **消息限制**: 非互相关注用户限制私信数量，防止骚扰
8. **反规范化设计**: users 表预计算统计字段，避免频繁 COUNT 查询
9. **智能刷新**: 私信列表变化检测，提升用户体验
10. **分类管理**: 支持大类小类灵活配置
