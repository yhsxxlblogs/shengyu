# 声愈项目面试核心问题文档

## 一、项目概述类问题

### Q1: 请介绍一下"声愈"项目是什么？解决了什么问题？

**参考答案:**
声愈是一个动物声音社交应用，核心功能包括：
1. **声音录制与分享** - 用户可以录制动物声音并分享
2. **社区互动** - 发布帖子、点赞、评论、关注
3. **实时私信** - WebSocket实现的即时通讯
4. **后台管理** - 内容审核、轮播图管理、通知发布

解决的需求：为动物爱好者提供一个声音分享和社交的平台，满足用户对动物声音内容的需求和社交互动的需求。

技术亮点：uni-app跨平台、原生WebSocket实现、Redis缓存优化、数据库反规范化设计。

---

### Q2: 项目的技术架构是怎样的？为什么选择这套技术栈？

**参考答案:**
**前端:** uni-app + Vue 3
- 选择原因：一套代码多端运行（H5、小程序、App），开发效率高

**后端:** Node.js + Express + MySQL + Redis
- Node.js：异步非阻塞，适合I/O密集型应用
- Express：轻量灵活，中间件丰富
- MySQL：关系型数据库，数据一致性要求高
- Redis：缓存热点数据，提高读取性能

## 二、前端技术问题

### Q3: uni-app是如何实现跨平台的？原理是什么？

**参考答案:**
uni-app基于Vue.js，通过编译器将Vue代码编译成不同平台的代码：
- **H5**: 编译成标准HTML/CSS/JS
- **小程序**: 编译成各平台小程序语法（WXML/WXSS等）
- **App**: 编译成原生渲染代码（使用Weex引擎或WebView）

运行时通过条件编译（`#ifdef`）处理平台差异，如：
```javascript
// #ifdef APP-PLUS
plus.android.requestPermissions(['android.permission.RECORD_AUDIO'])
// #endif
```

### Q4: 项目中WebSocket是如何封装的？如何保证重连？

**参考答案:**
封装在`websocket.js`中，主要功能：
1. **连接管理**: 维护连接状态，自动重连
2. **心跳检测**: 定时发送ping/pong保活
3. **消息队列**: 断线期间消息缓存，重连后发送
4. **事件订阅**: 支持多组件监听同一事件

重连策略：
- 指数退避算法，最大重试间隔30秒
- 最多重试10次，失败后提示用户
- 网络恢复时自动重连

## 三、后端技术问题

### Q7: 项目中JWT是如何实现的？如何防止Token被盗用？

**参考答案:**
**实现方式:**
```javascript
// 生成Token
const token = jwt.sign({ id: user.id }, 'secret_key', { expiresIn: '1d' })

// 验证Token
jwt.verify(token, 'secret_key', (err, decoded) => {
  if (err) return res.status(403).json({ error: 'token无效' })
  req.user = decoded
  next()
})
```

**安全措施:**
1. 设置合理的过期时间（1天）
2. 使用HTTPS传输
3. Token存储在客户端storage，不暴露在URL
4. 敏感操作二次验证
5. 支持Token失效（用户退出登录）

### Q8: 数据库为什么选择反规范化设计？优缺点是什么？

**参考答案:**
**反规范化设计:**
在users表添加冗余字段：posts_count, likes_received_count, followers_count等

**优点:**
- 减少JOIN查询，提高读取性能
- 统计数据查询从O(n)变为O(1)
- 适合读多写少的场景

**缺点:**
- 数据冗余，占用更多存储
- 需要维护数据一致性
- 写入性能略有下降

**维护策略:**
- 使用数据库触发器或应用层更新统计字段
- 定期全量校准统计数据

---

### Q9: Redis缓存在项目中是如何应用的？缓存穿透、击穿、雪崩如何解决？

**参考答案:**
**应用场景:**
- 帖子列表、热门声音、用户统计等热点数据
- 缓存时间5-10分钟

**缓存问题解决方案:**

1. **缓存穿透**（查询不存在的数据）
   - 解决方案：布隆过滤器或缓存空值

2. **缓存击穿**（热点key过期）
   - 解决方案：互斥锁，只有一个线程去加载数据

3. **缓存雪崩**（大量key同时过期）
   - 解决方案：过期时间加随机值，避免同时失效

### Q11: 文件上传是如何限制大小和类型的？如何防止恶意文件上传？

**参考答案:**
**multer配置:**
```javascript
const upload = multer({
  storage: storage,
  limits: { fileSize: 5 * 1024 * 1024 }, // 5MB
  fileFilter: function (req, file, cb) {
    const filetypes = /jpeg|jpg|png|gif/
    const extname = filetypes.test(path.extname(file.originalname).toLowerCase())
    const mimetype = filetypes.test(file.mimetype)
    if (extname && mimetype) {
      return cb(null, true)
    }
    cb(new Error('只允许上传图片文件'))
  }
})
```

**安全措施:**
1. 限制文件扩展名和MIME类型
2. 限制文件大小
3. 文件重命名（避免特殊字符）
4. 存储在非Web目录，防止直接执行
5. 图片压缩处理

### Q22: 如何设计一个高并发的点赞系统？

**参考答案:**

**方案:**
1. **写入优化** - 使用消息队列异步处理
2. **读取优化** - Redis缓存点赞数
3. **去重** - Redis Set存储用户点赞记录

**实现:**
```javascript
// 点赞请求入队
await redis.lpush('like:queue', JSON.stringify({ postId, userId }))

// 消费队列更新数据库
while (true) {
  const task = await redis.brpop('like:queue', 0)
  const { postId, userId } = JSON.parse(task[1])
  // 更新数据库
  await db.query('UPDATE posts SET like_count = like_count + 1 WHERE id = ?', [postId])
}
```
